> **Author packages on your workstation**, not on target VMs. PowerShell 7 is cross-platform â€” you can build Windows packages from any OS.

### 1. Create the Package


This installs `GuestConfiguration` and the required DSC modules (`PSDscResources`), creates a deployable .zip from the pre-generated MOF, and runs a local compliance test.

### 2. Upload to Azure Blob Storage

The container can stay private â€” the SAS token provides read access for the GC agent.

```powershell

Set-AzStorageBlobContent -Container 'guestconfiguration' \
  -File '.\output\WinRegistryAuditManual.zip' -Context $ctx

  -Blob 'WinRegistryAuditManual.zip' -Permission r \
  -ExpiryTime (Get-Date).AddYears(3) -Context $ctx -FullUri
```

### 3. Get the Content Hash

```powershell

```

### 4. Create and Assign the Policy

1. Open `policy.json` and replace `{{contentUri}}` with the SAS URL and `{{contentHash}}` with the hash
2. Create the policy definition:

```powershell
New-AzPolicyDefinition -Name 'WinRegistryAuditManual' -Policy '.\policy.json' -Mode 'Indexed'
```

3. Assign it:

```powershell

New-AzPolicyAssignment -Name 'WinRegistryAuditManual' -PolicyDefinition $def \
  -Scope '/subscriptions/<subscription-id>'
```
